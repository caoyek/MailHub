帮我实现 MailHub「信驿」后台管理系统，Next.js 14 App Router + TypeScript + Tailwind CSS + shadcn/ui。

## 设计理念
像一本精心维护的驿站台账：
- 纸张底色（#F5EDD6），不是白色，不是深色
- 信息用台账行的方式呈现，横线分隔，纵列对齐
- 朱砂红（#C0392B）只出现在状态标记、主按钮、数字高亮
- 没有渐变卡片，没有彩色图标，没有现代 SaaS 的圆润感
- 整体像一份被认真填写的公文账册

## 设计规范
（同首页规范完全一致，颜色命名也保持一致）
- 底色：#F5EDD6（油皮纸）
- 深墨：#1C1410
- 次要文字：#6B5A45
- 强调（朱砂红）：#C0392B
- 卡片/表面：#FDF6E4
- 边框/分隔：#DDD0B3
- 成功：#2D6A4F（沉稳墨绿，不用鲜绿）
- 失败：#C0392B（复用朱砂红）
- 字体：Noto Serif SC（标题/标签）+ Noto Sans SC（正文/数据）+ JetBrains Mono（时间戳/数值）

## 技术要求
- /app/dashboard/layout.tsx 包含 Sidebar + Topbar
- 所有颜色在 tailwind.config.ts 定义（语义命名：paper / ink / vermilion / scroll 等）
- /lib/api.ts 统一封装请求
- /hooks/useLiveStream.ts 封装 WebSocket，自动重连（指数退避）
- next/font 引入 Noto Serif SC + Noto Sans SC + JetBrains Mono（同首页）
- 字体在 layout.tsx 引入一次，通过 CSS 变量传递给 Tailwind（font-serif / font-sans / font-mono）
- 印章组件 SealLogo.tsx 支持 variant（dark / light）参数，深色背景用 light 版本
- 所有表单 react-hook-form + zod
- mock 数据 /lib/mock.ts，接口 URL 从 NEXT_PUBLIC_API_URL 读取
- 响应式：移动端 Sidebar 改为顶部 Drawer，表格横向滚动
- 无任何渐变、发光、粒子、圆润卡片等现代 SaaS 视觉套路
- shadcn/ui 组件保留功能逻辑，但视觉上通过 className 完全覆盖为台账风格
