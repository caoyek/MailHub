你是一个全栈架构师。帮我搭建一个叫「信驿 MailHub」的邮件聚合转发系统，完整的 monorepo 项目结构。

## 技术栈要求

后端：
- Python 3.11+
- FastAPI（API 服务 + WebSocket 实时推送）
- APScheduler（IMAP 定时轮询）
- 数据存储：开发阶段全部用 JSON 文件，不引入任何数据库和 ORM
  - data/logs.json       存储邮件处理日志
  - data/rules.json      存储路由规则
  - data/channels.json   存储渠道配置
  - data/stats.json      存储每日统计汇总
  - 所有读写封装在 src/store.py 中，对外暴露统一的增删改查方法
  - 写入时加文件锁（fcntl 或 threading.Lock）防并发冲突
- python-dotenv（密钥管理）+ YAML（路由规则的人工可编辑备份）
- 渠道模块用抽象基类设计（BaseChannel），已实现：邮箱转发、企微群机器人 Webhook
- httpx（异步 HTTP 请求）
- imap-tools（IMAP 邮件拉取）

前端：
- Next.js 14（App Router）
- TypeScript
- Tailwind CSS
- shadcn/ui 组件库
- Framer Motion（动画，仅入场动效）
- Recharts（数据图表）
- next-auth（登录鉴权，credentials provider）
- next/font 引入 Noto Serif SC + Noto Sans SC + JetBrains Mono

部署（开发阶段）：
- 直接 python main.py 启动后端
- next dev 启动前端
- 不需要 Docker，不需要 Redis，不需要任何外部服务
- 只需要一个 .env 文件配置邮箱和 Webhook 信息

NEXT_PUBLIC_API_URL=http://localhost:8000

## 其他要求

- data/ 目录加入 .gitignore，但保留 data/.gitkeep 占位
- 所有 JSON 文件初始化为空数组 [] 或空对象 {}，后端启动时自动检测并创建
- store.py 的写操作使用 threading.Lock 保证并发安全
- 后端认证极简：单用户，账号密码写在 .env，
  登录接口 POST /api/auth/login 返回 JWT token，
  其余接口用 Bearer token 校验，deps.py 封装校验逻辑
- 前端 next-auth 使用 credentials provider，
  调用后端 /api/auth/login 验证，不需要数据库 session
- 输出完整目录树和每个文件的单行职责注释，不需要写代码内容
- 最后附上本地开发启动步骤（5 步以内）
